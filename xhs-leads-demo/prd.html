意向判断逻辑（冷 / 温 / 热）
私信通 AI 策略层 · 每轮对话后实时更新评分
评分模型（旅游行业）
信号类型具体行为（旅游场景示例）加分
明确意图主动给出出行日期（「五一想去」「6月下旬两周」）+30
询问具体产品库存/可用档期（「三亚5天有团吗」）+20
询问价格/套餐报价（「人均多少」「含机票吗」）+20
主动比价或要求优惠（「能便宜点吗」「有没有优惠券」）+25
询问支付/定金/合同方式（「怎么预订」「定金多少」）+30
互动深度连续对话 ≥ 5 轮+15
对 AI 推送内容有实质回复（非「好的」「嗯」）+10
主动追问细节（「有单人房吗」「导游是本地人吗」）+10
衰减因子最后回复 > 24h×0.8
最后回复 > 72h×0.5
明确拒绝（「不需要了」「太贵了不去」）且无后续×0.3
意向等级定义
🔴 热意向 ≥ 75 分
判断逻辑：满足以下至少一项 + 近 24h 内有互动
• 主动给出明确出行日期
• 询问价格/产品细节（问价即热信号）
• 发起价格谈判或询问预订方式
运营动作：立即推送给销售，标红置顶；超 30min 未接管触发 push 提醒
🟡 温意向 40 ~ 74 分
判断逻辑：有明确咨询意图（问目的地/行程/攻略）但未触及价格/档期；或高分但沉默 24~72h
运营动作：AI 继续跟进；24h 无回复触发唤醒话术；3 次无回复转销售人工判断
⚪ 冷意向 < 40 分
判断逻辑：仅随意浏览/泛问，无实质需求信号；或沉默 > 72h 且无历史高分记录
运营动作：进入沉默池；7天自动唤醒一次；30天无互动归档「已关闭」
与投放侧对齐：意向标签传递给知然归因模型时使用枚举值（hot / warm / cold），不传原始分值，避免销售对数字产生误解。标签定义一次对齐后，PM-2（工作台侧）与 PM-3（投放侧）独立推进。

回传填报
闭环飞轮的关键环节 · 结果回传 → 投放归因 → 更精准线索
① 入口设计（两处，职责分离）
入口 A · 工作台右侧面板（主入口）
触发条件：点击「一键接管」后解锁
逻辑：先跟进，后回传，强制顺序
适合：正在跟进的线索，即时填写
入口 B · 线索列表行内操作（补录入口）
触发：列表行「···」→「填写结果」
展示：底部轻量 Sheet，3 秒内完成
适合：线下跟进后回来补录
② 状态流转与填写逻辑
跟进中（平台自动同步） └─ 用户预约 → 「已预约」状态（平台/用户自主） └─ 用户到店 → 「是否到店」回传（未到店/已到店） └─ 已到店 → 「是否成交」回传（未成交/已成交） └─ 未成交/已成交 → 线索完成归档
数据来源分层：
● 跟进中：平台站内行为自动识别，无需用户填写
● 已预约：平台可获取的预约行为（商家系统内预约/用户自主预约）
● 是否到店/是否成交：平台无法直接获取的线下行为，必须依赖商家回传
填写项类型展示条件说明
跟进中状态标签AI/人工正在跟进时自动展示平台自动同步，无需手动填写
已预约状态标签检测到用户预约行为时展示平台可获取则自动显示，也可手动标记
是否到店（必填）单选始终展示❌ 未到店 / ✅ 已到店
是否成交（必填）单选始终展示❌ 未成交 / 🎉 已成交
成交金额数字输入选「已成交」时展示单位：元；用于 ROI 归因
备注自由文本始终展示，选填≤ 100 字；Placeholder：如「本周五来看行程」「太贵了暂时放弃」
③ 填写激励提示（提交后展示）
💡 你的回传让 AI 更聪明
本次数据已同步至投放模型，系统将识别「这类用户特征」，在下一批广告中优先触达相似的高意向用户。
本月你已回传 31 条，预估有效线索率提升 +8%。
文案策略：数字要具体化（31条、+8%），而非「帮助 AI 优化」这类虚泛表达；归因到「这个线索」让销售感受到个体贡献；每月初重置计数，形成周期正向反馈。
④ 数据流向（与知然/投放团队对齐的字段格式）
{ "event": "lead_feedback", "lead_id": "L20260501_YUNNAN_001", "user_id": "xhs_uid_xxxxxx", "intent_level": "hot", // hot | warm | cold "feedback_status": "visited", // visited | closed | following | invalid "deal_amount": null, // 成交时填写，单位：分 "next_followup_at": null, // ISO8601，跟进中时填写 "note": "用户五一档期感兴趣，本周五来门店看行程单", "feedback_by": "sales_id_xxxx", "feedback_at": "2026-04-02T20:00:00+08:00", "source_ad_id": "ad_xxxxxx", // 归因到广告计划 "source_skill": "套餐报价" // 最后触发的 Skill }
核心归因字段：lead_id + intent_level + feedback_status + source_ad_id → 知然归因模型 → 投放策略闭环优化

线索状态流转
Tab 栏状态映射 · 自动流转规则 · 数据来源分层
① 数据获取分层逻辑
🟢 平台自动同步
跟进中：站内互动行为自动识别
无需商家手动填写，实时更新状态
🟡 平台可获取
已预约：系统内预约/用户自助预约
有数据则自动展示，也可手动补录
⚪ 需商家回传（线下行为）
是否到店、是否成交：平台无法感知的线下行为
必须通过商家主动回传，是投放归因的核心数据
② 完整状态流转
新进线索 └─ 跟进中（平台自动：站内互动中） └─ 用户预约 → 已预约（平台/用户自主） ├─ 用户未到门店 → 未到店（商家回传） │ └─ 成交/未成交 → 已归档 └─ 用户已到门店 → 已到店（商家回传） ├─ 填写「已成交」→ 已成交 → 归档 └─ 填写「未成交」→ 未成交 → 归档
③ Tab 栏与状态映射
Tab 名称包含状态排序规则
处理中跟进中（AI/人工）+ 已预约热线索优先，最新互动时间倒序
已接管销售已接管的线索（待填写结果）接管时间倒序
已完成已到店/未到店 + 已成交/未成交完成时间倒序
关键设计原则：
1. 数据分层透出：平台能拿的自动展示（减轻商家负担），拿不到的明确引导回传
2. 渐进式填写：「到店」→「成交」两阶段，避免一次填写过多字段
3. 离线可补录：支持事后回来填写，不强制实时提交
